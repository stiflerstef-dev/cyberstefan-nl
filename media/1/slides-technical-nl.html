Sau

Technische Deep-Dive

🖥️ Overzicht Sau

  • HackTheBox machine: Sau
  • Moeilijkheidsgraad: Easy
  • Platform: Linux (Ubuntu-based)

🔍 Recon & Enumeratie

  • Nmap-scan: poorten 22 (OpenSSH 8.2p1), 80 (gefilterd HTTP), 55555 (open)
  • Service op 55555: Golang HTTP server – request‑baskets v1.2.1
  • Gefilterde poort 80 wijst op interne webdienst alleen bereikbaar via SSRF

💥 Initiele Toegang

  • SSRF‑kwetsbaarheid CVE-2023-27163 in request‑baskets (forward_url naar http://127.0.0.1:80)
  • Interne service geïdentificeerd als Mailtrail v0.53
  • Ongeauthenticeerde OS command injection via username‑parameter → reverse shell als gebruiker puma

⬆️ Privilege Escalatie

  • sudo -l toonde NOPASSWD voor /usr/bin/systemctl status trail.service
  • systemctl status start pager (less) als root bij lange output
  • In less: !sh geeft root‑shell → toegang tot root.txt

🛠️ Tools & Technieken

  • nmap -sC -sV -p- voor poortdetectie
  • curl om SSRF basket te maken en interne endpoint te benaderen
  • python3 -m http.server 8000 om shell.sh te serveren
  • nc -lvnp 9001 als Netcat listener voor reverse shell
  • sudo systemctl status trail.service gevolgd door !sh in less

📚 Geleerde Lessen

  • SSRF kan gefilterde, interne diensten blootstellen – netwerkfiltering alleen is niet voldoende
  • Tijdelijk patchen van derde‑partijcomponenten (request‑baskets, Mailtrail) breekt de aanvalsketen
  • Sudo‑toegang tot pagers zoals less biedt onbedoelde root‑toegang; gebruik LESSSECURE=1 of vervang pager
  • Defense in depth: elke enkele zwakke schakel (SSRF → command injection → sudo misconfig) zou de compromittering hebben voorkomen
