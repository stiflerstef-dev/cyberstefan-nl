Support

Technical Deep-Dive

🖥️ Overview

  • Machine: Support (HackTheBox)
  • Platform: Windows, Easy difficulty
  • Key services: SMB (445), LDAP/AD (389/636), WinRM (5985)
  • Objective: obtain user.txt and root.txt

🔍 Recon & Enumeration

  • Anonymous SMB share \\10.129.43.78\support-tools accessible
  • Retrieved UserInfo.exe.zip, extracted and decompiled .NET binary with ILSpy
  • Found encoded password and XOR key; decrypted to LDAP credentials ldap@support.htb / nvEfEK16^1aM4$e7AclUf8x$tRWxPWO1%lmz
  • LDAP query revealed support user's 'info' field containing plaintext password Ironside47pleasure40Watchful
  • Confirmed AD domain support.htb and user privileges

🚪 Initial Access

  • Used LDAP credentials to bind and retrieve support user password from AD attribute
  • Logged into target via evil-winrm as support user with password Ironside47pleasure40Watchful
  • Collected user.txt from C:\Users\support\Desktop\user.txt
  • Established foothold with standard user privileges

🔐 Privilege Escalation (RBCD)

  • Enumerated privileges: SeMachineAccountPrivilege enabled, member of Shared Support Accounts (SID …1103)
  • PowerView showed Shared Support Accounts has GenericAll over the domain controller
  • Created fake machine account FakeComputer2 using Powermad, set msDS-AllowedToActOnBehalfOfOtherIdentity via RawSecurityDescriptor
  • Generated correct NTLM hash for Password123!, forged S4U ticket with Rubeus to impersonate Administrator
  • Converted ticket to ccache, used impacket-secretsdump to extract Administrator NT hash, then pass-the-hash via evil-winrm for root shell

🛠️ Tools & Techniques

  • SMB enumeration: smbclient (anonymous), unzip
  • .NET analysis: ILSpy (decompilation), base64 + Python XOR decryption
  • AD enumeration: ldapsearch with discovered credentials
  • PowerShell modules: PowerView.ps1 (ACL enumeration), Powermad.ps1 (machine account creation)
  • Kerberos abuse: Rubeus.exe (S4U), impacket-ticketConverter & secretsdump
  • Shell access: evil-winrm (plaintext & pass-the-hash)

💡 Lessons Learned

  • Never embed credentials in compiled .NET binaries; they are trivially recovered via decompilers
  • Avoid storing secrets in AD user attributes (e.g., 'info') – readable by any authenticated LDAP user
  • SeMachineAccountPrivilege combined with GenericAll over a DC enables full domain compromise via Resource‑Based Constrained Delegation
  • Rubeus‑generated Kerberos tickets are not forwarded by evil-winrm for SMB; use impacket on the attacker host for ticket‑based operations
  • Mitigate by restricting machine account creation, auditing privileged groups, enforcing least privilege, and monitoring anomalous RBCD configurations
