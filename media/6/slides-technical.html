Support

Technical Deep-Dive

🖥️ Overview

  • Machine: Support (HTB, Easy difficulty)
  • Platform: Windows domain controller with SMB, LDAP, WinRM services
  • Initial foothold: anonymous SMB share revealing a .NET binary
  • Privilege path: LDAP credentials → support user → SeMachineAccountPrivilege → GenericAll over DC → RBCD attack
  • Outcome: Administrator shell and root flag

🔍 Recon & Enumeration

  • nmap -sC -sV -oN support.nmap 10.129.43.78 → open ports 139/445 (SMB), 389 (LDAP), 5985 (WinRM)
  • SMB enumeration via smbclient -L //10.129.43.78 -N → share 'support-tools' accessible
  • Share contains UserInfo.exe.zip → downloaded and extracted
  • No credentials required for SMB null session
  • LDAP service reachable, enabling later authenticated queries

🔑 Initial Access

  • Decompiled UserInfo.exe with ILSpy → revealed XOR‑encrypted LDAP password for user 'ldap'
  • Decrypted password via Python script (XOR with key 'armando' and 0xDF)
  • Authenticated LDAP bind as ldap@support.htb → queried AD for user 'support'
  • Found support password stored in the AD 'info' field (cleartext)
  • Logged in via evil-winrm as support → obtained user.txt

⬆️ Privilege Escalation

  • whoami /all shows SeMachineAccountPrivilege and membership in 'Shared Support Accounts'
  • PowerView.ps1 reveals the group has GenericAll rights over the Domain Controller object
  • Powermad.ps1 creates a controlled machine account (FakeComputer)
  • Configured RBCD on DC: set msds-allowedtoactonbehalfofotheridentity to FakeComputer’s SID
  • Rubeus.exe S4U obtained a Kerberos ticket for Administrator; impacket/secretsdump extracted Administrator NTLM hash; evil-winrm Pass‑the‑Hash gave admin shell

🛠️ Tools & Techniques

  • Scanning: nmap, smbclient
  • Binary analysis: ILSpy (Linux), unzip
  • Password recovery: Python XOR decryption script
  • LDAP queries: ldapsearch
  • Shell: evil-winrm (WinRM), PowerView.ps1, Powermad.ps1, Rubeus.exe
  • Kerberos & hash extraction: impacket (ticketConverter, secretsdump), Pass-the-Hash

💡 Lessons Learned

  • .NET binaries are fully decompilable → never store secrets (even XOR‑encrypted) in compiled code
  • AD attributes (e.g., info field) are readable by any authenticated user → avoid placing passwords there
  • SeMachineAccountPrivilege combined with GenericAll over a DC enables RBCD‑based domain compromise
  • Special characters ($) in passwords require single quotes in bash to prevent variable expansion
  • Kerberos tickets imported via Rubeus /ptt do not work with evil-winrm for SMB; use impacket on your host
  • Machine accounts on HTB may be reset; re‑create them in each new session
